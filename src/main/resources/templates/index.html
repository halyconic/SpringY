The Y combinator

Many people are familiar with the company Y combinator. Fewer people know this company is actually based on a mathematical concept called the fixed point combinator. The Y combinator is the most succinct way to implement recursion in lambda calculus.

If we remove the ability to write recursive code in Java, we can still write code in a style that is not recursive, but then apply the Y combinator to it to make it recursive.

The second concept I will use is memoization. This allows us to trade memory for time, with the optimization divorced from the business logic. Unlike typical caching patterns, the function inputs (or cache keys) are invulnerable to key conflicts that might arise when business requirements change.

I combine these two concepts to implement the fibonacci sequence. The sequence goes as follows: 1, 1, 2, 3, 8... where each number is the sum of the previous two numbers, where f(4) = 8. Naturally, when calculating f(4), you must recalculate all previous numbers, so caching is useful here.
